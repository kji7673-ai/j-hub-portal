부록

따라하기 가이드

5개 플랫폼을 직접 만들어보는 실전 워크북

이 장은 책을 읽고 "나도 해보고 싶다"는 분을 위한 실전 가이드입니다. 각 플랫폼별로 첫 30분 안에 결과물을 볼 수 있는 최소 단계만 정리했습니다. 코드를 쓸 필요가 없습니다. AI에게 아래의 프롬프트를 그대로 복사·붙여넣기만 하면 됩니다.

시작 전 준비물
  • AI 도구: Google Gemini, ChatGPT, Claude 중 하나 (유료 버전 권장)
  • 브라우저: Chrome 또는 Safari
  • 파일 관리: 바탕화면에 "AI프로젝트" 폴더 하나 만들기
  • 마음가짐: 에러가 나면 복사해서 AI에게 던진다. 당황하지 않는다.

플랫폼 ① — 업무 자동화 스크립트 (CAD 자동배치 사례)

난이도: ★★☆☆☆ | 소요 시간: 30분 | 필요 도구: AI + 업무에 쓰는 프로그램 (CAD, Excel, 한글 등)

Step 1. 반복 업무를 한 문장으로 정의한다

당신의 회사에서 가장 반복적이고 시간이 많이 걸리는 업무를 떠올리세요. 그리고 아래 형식으로 한 문장을 만드세요:

"[프로그램 이름]에서 [반복 작업]을 자동으로 해주는 [스크립트/매크로]를 만들어줘."
👤 프롬프트 예시 (건축사 사무소)
"AutoCAD에서 사각형 대지 안에 아파트 건물 3개를 자동으로 배치하고, 건물 간 이격 거리가 10m 이상인지 확인하는 LISP 스크립트를 만들어줘."
👤 프롬프트 예시 (제조업)
"Excel에서 매달 입력하는 월간 매출 데이터를 자동으로 그래프로 만들고, 전월 대비 변동률을 계산하는 VBA 매크로를 만들어줘."

Step 2. AI의 결과물을 받아 실행한다

AI가 코드를 줄 것입니다. 그 코드를 그대로 복사하여 해당 프로그램에 붙여넣으세요.

  • AutoCAD: 명령줄에 APPLOAD 입력 → AI가 준 .lsp 파일 로드
  • Excel: Alt+F11 → 새 모듈 → AI가 준 VBA 코드 붙여넣기 → 실행
  • 한글(HWP): 도구 → 매크로 → AI가 준 코드 붙여넣기

Step 3. 에러가 나면? (문과형 디버깅)

에러가 날 확률은 높습니다. 당황하지 마세요. 에러 메시지를 복사하여 AI에게 이렇게 보내세요:

"이 에러가 났어. 뭐가 문제야? [에러 메시지 붙여넣기]"

AI가 수정된 코드를 줄 것입니다. 이것을 2~3번 반복하면 대부분 작동합니다.

Step 4. 완성 확인

  • 스크립트가 에러 없이 실행되는가?
  • 결과물이 기대한 것과 일치하는가?
  • 수동으로 했을 때와 결과가 같은가? (반드시 수동 교차 검증!)
[그림 4] AI가 자동 생성한 현황 삽도 — 역삼동 743번지 일원. VWorld + OSM 데이터를 결합하여 건물 외곽(빨간색), 학교, 공원, 종교시설, 지하철역을 자동으로 표시한다. "(주)진양 엔지니어링 건축사사무소" 명의가 좌측 하단에 자동 삽입된다.

플랫폼 ② — 자동 보고서 생성기 (통합검토보고서 사례)

난이도: ★★★☆☆ | 소요 시간: 1시간 | 필요 도구: AI + 보고서 양식

Step 1. 기존 보고서 양식을 AI에게 보여준다

당신이 매번 작성하는 보고서(사업 계획서, 견적서, 검토서 등)의 빈 양식이나 이전 작성 사례를 AI에게 보여주세요. 사진을 찍어서 붙여넣어도 됩니다.

👤 프롬프트 예시
"이 보고서 양식을 보고 똑같은 구조로 HTML 파일을 만들어줘. 나중에 데이터만 바꿔 넣으면 자동으로 보고서가 생성되게 해줘. [이미지 첨부]"

Step 2. 데이터 조사를 AI에게 시킨다

👤 프롬프트 예시 (건축)
"서울시 용산구 용문동 일대의 2024년 기준 개별공시지가 평균을 조사해줘. 출처도 알려줘."
👤 프롬프트 예시 (일반 기업)
"우리 업종(식품 제조업)의 2024년 평균 영업이익률을 조사해줘. 한국은행, 통계청 등 신뢰할 수 있는 출처로."

Step 3. 환각 검증 — 반드시 수동 확인!

⚠️ 가장 중요한 단계 — AI 환각 검증
  • AI가 제시한 수치를 원본 출처에서 직접 확인한다 (law.go.kr, KOSIS 등)
  • "이 숫자가 말이 되나?"를 당신의 업종 경험으로 판단한다
  • 의심스러우면 "이 데이터를 어디서 가져왔어? 출처 URL을 줘"라고 물어본다
  • 같은 질문을 다른 표현으로 한 번 더 물어본다. 답이 달라지면 환각이다

Step 4. PDF로 출력한다

AI가 만든 HTML 파일을 Chrome 브라우저로 열고, Ctrl+P → PDF로 저장을 누르면 깔끔한 보고서 PDF가 생성됩니다. 이것을 거래처나 고객에게 제출하면 됩니다.

Step 5. 반복 — v1에서 v48까지

"표가 깨져 있어, 고쳐줘" → "폰트가 너무 작아" → "여기에 도면을 추가해줘" — 이런 피드백을 한 번에 하나씩 보내세요. AI는 5분마다 수정본을 줍니다. 사람에게 시키면 3일 걸리는 수정이, AI에게는 하루에 33번 가능합니다.

플랫폼 ③ — 사내 디지털 미디어 (진양저널 사례)

난이도: ★★☆☆☆ | 소요 시간: 45분 | 필요 도구: AI + USB(또는 공유 폴더)

Step 1. 한 문장으로 요구한다

👤 프롬프트 (그대로 복사하세요)
"우리 회사 직원들이 사내 소식을 읽을 수 있는 게시판 형태의 웹페이지를 만들어줘. 글을 올릴 수 있고, 댓글을 달 수 있고, 읽은 사람 수가 표시되게 해줘. HTML 파일 하나로 만들어줘."

Step 2. 첫 글을 작성한다

AI가 준 HTML 파일을 Chrome으로 열면 게시판이 보입니다. 직접 첫 번째 글을 작성해보세요:

"이번 주 주간 회의 요약", "신규 프로젝트 안내", "여름 휴가 일정" 등

Step 3. 배포한다

  • 방법 A (USB): HTML 파일을 USB에 담아 각 PC에 복사. 더블클릭하면 브라우저에서 열림
  • 방법 B (공유 폴더): 회사 NAS나 공유 폴더에 올리고, 전직원에게 링크 공유
  • 방법 C (서버): "이 페이지를 Flask 서버로 만들어줘"라고 AI에게 추가 지시. 사내 IP로 접속 가능

Step 4. 게이미피케이션 추가 (선택)

직원들이 안 읽는 문제를 해결하려면:

👤 추가 프롬프트
"글을 읽으면 포인트가 쌓이고, 댓글을 달면 추가 포인트를 주는 게이미피케이션을 추가해줘. 상위 5명의 랭킹을 메인 페이지에 보여줘."

플랫폼 ④ — 사내 문서 검색 시스템 (DocReview 사례)

난이도: ★★★★☆ | 소요 시간: 1~2시간 | 필요 도구: AI + Python + 검색할 PDF 파일들

Step 1. 준비

이 플랫폼은 다른 것보다 약간 복잡합니다. 먼저 AI에게 환경을 세팅하도록 시키세요:

👤 프롬프트 (환경 세팅)
"내 컴퓨터에 Python이 설치되어 있어. PDF 파일들을 업로드하면 내용을 검색할 수 있는 웹 페이지를 만들고 싶어. 필요한 패키지 설치부터 차근차근 알려줘."

Step 2. PDF 업로드 기능

👤 프롬프트
"PDF를 드래그 앤 드롭으로 업로드하면, OCR로 텍스트를 추출해서 검색 가능하게 만들어줘. Flask로 웹 서버를 만들어줘."

Step 3. 검색 기능

👤 프롬프트
"검색창에 키워드를 입력하면, 어떤 PDF의 몇 페이지에 그 내용이 있는지 보여줘. 해당 문장을 하이라이트해서 표시해줘."

Step 4. 테스트

  • 실제 업무 문서 3~5개를 업로드한다
  • 알고 있는 내용을 검색하여 올바른 결과가 나오는지 확인한다
  • 수동 검색(Windows 탐색기)과 결과를 비교한다

플랫폼 ⑤ — 사내 교육 포털 (J-Hub 사례)

난이도: ★★★☆☆ | 소요 시간: 2시간 | 필요 도구: AI + 교육할 콘텐츠

Step 1. 교육 콘텐츠를 먼저 만든다

포털을 만들기 전에, 교육할 내용이 있어야 합니다. AI에게 이렇게 시키세요:

👤 프롬프트 (콘텐츠 생성)
"우리 회사 직원들이 AI를 업무에 활용하는 방법을 배울 수 있는 교육 자료를 만들어줘. 초급·중급·고급으로 나누고, 각 단계별로 3개씩 총 9개 교육 모듈을 만들어줘. 각 모듈은 설명 + 실습 예제 + 퀴즈로 구성해줘."

Step 2. 포털 틀을 만든다

👤 프롬프트 (포털 생성)
"위의 9개 교육 모듈을 목록으로 보여주는 웹사이트를 만들어줘. 로그인 기능, 진도 추적(어디까지 읽었는지), 퀴즈 점수 저장 기능을 넣어줘. Flask + SQLite로 만들어줘."

Step 3. 직원 등록

👤 프롬프트
"관리자(나)가 직원을 등록할 수 있는 관리자 페이지를 만들어줘. 직원 이름과 초기 비밀번호를 입력하면 계정이 생성되게 해줘. 비밀번호는 암호화해서 저장해줘."

Step 4. 보안 점검 (레드팀)

교육 포털이 돌아가기 시작하면, 반드시 레드팀을 돌리세요:

👤 프롬프트 (레드팀 — 그대로 복사하세요)
"지금 만든 시스템의 보안을 해커 관점에서 감사해줘. 아래 항목을 체크해줘: 1. 비밀번호가 암호화되어 저장되는가? 2. 로그인을 무한 반복 시도할 수 있는가? (Brute-force) 3. 입력란에 악성 코드를 넣을 수 있는가? (XSS) 4. URL을 조작하여 다른 사람의 데이터에 접근할 수 있는가? 5. 소스 코드나 데이터베이스 파일에 직접 접근할 수 있는가? 심각도를 Critical / High / Medium / Low로 분류하고, 각각의 수정 방법을 알려줘."

Step 5. 완성 확인

  • 본인 계정으로 로그인이 되는가?
  • 교육 문서를 클릭하면 내용이 표시되는가?
  • 진도가 저장되는가? (로그아웃 후 재로그인 시 유지 확인)
  • 퀴즈 점수가 기록되는가?
  • 보안 감사에서 Critical 취약점이 없는가?
따라하기의 핵심 원칙 3가지
원칙 1. 한 번에 하나만 시킨다

"로그인 기능 + 검색 기능 + 게시판 + 결제 시스템을 다 만들어줘"라고 하지 마세요. "로그인 기능만 먼저 만들어줘." 이것이 작동하면 다음을 시킵니다.

원칙 2. 매 단계마다 직접 눈으로 확인한다

AI가 "완료했습니다"라고 해도 반드시 직접 테스트하세요. 브라우저를 열고, 버튼을 눌러보고, 데이터를 입력해보세요. AI의 "완료"를 믿지 말고 당신의 눈을 믿으세요.

원칙 3. v1이 나오면 즉시 레드팀을 돌린다

v1이 작동하는 순간, 기쁨에 취하지 말고 "이 시스템의 문제점을 냉정하게 5가지만 짚어줘"라고 시키세요. 이것이 v2를 만드는 가장 빠른 길입니다. 나도 이 방법으로 v1에서 v48까지 갔습니다.
