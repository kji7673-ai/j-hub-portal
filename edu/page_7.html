플랫폼 진화의 연대기: 수작업에서 AI 통합 플랫폼까지

끊임없는 발전, J-Hub 플랫폼의 진화 단계

현재 우리가 사용하고 있는 '정비사업 통합검토보고서'와 'J-Hub 교육 플랫폼'은 하루아침에 만들어진 것이 아닙니다. 기술의 발전과 레드팀의 치열한 비판, 그리고 실무진의 피드백이 결합되어 다음의 5단계를 거쳐 폭발적으로 진화해 왔습니다.

🚀 Phase 1: 태동기 (수작업과 엑셀의 시대)

모든 법규 검토와 대지면적 산출을 담당자의 경험과 수작업에 의존하던 시기입니다. 엑셀을 활용해 계산을 자동화하려 했으나, 잦은 휴먼 에러와 파편화된 데이터 관리로 인해 초기 검토에 막대한 시간이 소요되었습니다.

🚀 Phase 2: AI 도입 및 프롬프트 표준화 (개인화된 엣지)

생성형 AI(ChatGPT 등)를 실무에 도입하기 시작한 단계입니다. '마법의 프롬프트 템플릿'과 같은 표준화된 SOP(표준운영절차)를 구축하여 누구나 일정 수준 이상의 법규 요약과 배치를 AI에게 지시할 수 있게 되었습니다. 하지만 데이터의 입력 자체는 여전히 사람의 몫이었습니다.

🚀 Phase 3: 데이터 파이프라인 연동 (Open API 혁명)

가장 핵심적인 도약의 시기입니다. 서울시 공공데이터, VWorld 공간정보 API를 시스템에 직접 연동했습니다. 사람이 데이터를 입력하는 대신, 지번만 주어지면 AI가 지구단위계획, 지적, 도로, 학교 등의 방대한 데이터를 자동 추출하여 오차율 0%의 '정비사업 통합검토보고서'를 5초 만에 뱉어내는 파이프라인이 완성되었습니다.

🚀 Phase 4: J-Hub Portal 구축 및 UX 고도화 (현재)

강력해진 기능과 복잡해진 매뉴얼을 사내에 효율적으로 전파하기 위해, GitHub Pages 기반의 클라우드 플랫폼인 'J-Hub Portal (진양저널)'이 탄생했습니다.
특히 레드팀(Red Team)의 매서운 감사(Audit)를 거쳐, 스마트폰에서도 완벽하게 구동되는 PWA 모바일 UX, 다크모드, 직관적인 그리드 아이콘 등 Apple 수준의 세련된 UI를 갖춘 전사적 소통 허브로 고도화되었습니다.

🚀 Phase 5: 생애주기 통합 관리 (우리가 나아갈 미래)

앞으로는 단순한 초기 사업성 검토를 넘어, 계획 설계, 인허가, 실시 설계, 그리고 시공 후 유지관리까지 건축의 전 생애주기(Lifecycle) 데이터를 하나의 플랫폼에서 AI가 매니징하는 진정한 '1인 1플랫폼' 체제로 진화할 것입니다.
